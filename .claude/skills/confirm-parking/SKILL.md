# confirm-parking

ユーザーが research-parking の結果を自分の目で再確認した自治体について、`PLACE-DATA.md` の該当行のステータスを **`[△]` → `[○]`** に書き換える。

## 入力

- 市区町村名 (例: `中央区`、`千代田区`、`武蔵野市`)
- 1 回の実行で複数自治体を指定された場合は順に処理。

## 絶対ルール

- **`[○]` への書き換えはユーザー専権** — ユーザーが明示的にこのスキルを呼んだ時 (あるいは「丸にして」「○にして」「確認済み」等と発言した時) **のみ** 実行する。research-parking や他のスキルから自動的に呼び出さない。
- 現在のステータスが `[△]` 以外の場合は **書き換えを保留してユーザーに確認** する (詳細は下記「ステータス別の挙動」)。
- `apps/web/data/discount-research.json` に **対象自治体のエントリが 0 件** の場合は書き換えを拒否し、研究データが無いことを報告する。
- `[★]` や `[✓]` への昇格はこのスキルでは行わない (別の段階)。
- このスキルは `discount-research.json` を **変更しない**。あくまで PLACE-DATA.md のチェックボックスだけを更新する。

## ステータス別の挙動

| 現在 | 動作 |
|---|---|
| `[△]` | **そのまま `[○]` に書き換える** (正常系) |
| `[~]` | research-parking が未完了の可能性。「`[△]` を経ずに直接 `[○]` にしてよいか」をユーザーに確認 |
| `[ ]` | リサーチ未着手。「リサーチ前提のステータスだが本当に `[○]` でよいか」を確認 |
| `[○]` | 既に丸。「既に `[○]` ですが何もしなくてよいですか?」と確認 (冪等扱い可) |
| `[★]` / `[✓]` | 上位ステータスを下げる動作になるので **拒否**。理由を説明 |
| 該当市区町村が PLACE-DATA.md に存在しない | エラー報告 (タイポの可能性、または新規追加が必要か確認) |

## ワークフロー

### 1. 事前確認

- `PLACE-DATA.md` を `Grep` で検索し、対象市区町村の行と現在のステータス記号を取得。
- 同時に `apps/web/data/discount-research.json` を読み、対象自治体の `municipality` に一致するエントリ数をカウント。

### 2. 判定

- 上記「ステータス別の挙動」表に従う。`[△]` 以外なら、書き換える前に必ず一言ユーザーに確認する (回答済みなら進める)。
- エントリ数が 0 の場合は「research-parking で 1 件も登録されていません。`[○]` にしても利用者が見るデータが無い状態になりますがよろしいですか?」と確認。

### 3. 書き換え

- `Edit` で該当行 1 箇所のみを置換 (`replace_all: false`)。
- 行末の自治体コード `(13xxx)` は触らない。記号の中身だけを変える。

### 4. 報告

簡潔に:

1. 書き換え結果 (`旧記号 → ○`)
2. その自治体のエントリ件数 (`full` / `partial` の内訳があるなら添える)
3. 次にやりやすい自治体 (もし `[△]` のものが他にあれば 1〜2 件サジェスト) — 任意

## やってはいけないこと

- `[○]` を `[★]` や `[✓]` に格上げする (このスキルの範囲外)。
- `discount-research.json` の内容を変更する。
- ユーザーの明示的な指示なしに発火する (research-parking の最後で自動的に呼ぶ等)。
- TaskCreate などで作業を細切れに追跡する (このスキルは 1 ターンで完結する)。
